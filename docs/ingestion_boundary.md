# Ingestion Layer: Relationship with insurance-nlp-aws

## Boundary

```
insurance-nlp-aws                    insurance-ai-decision-workflow
─────────────────────────────────    ─────────────────────────────────────────
PDF documents                    →   (consumed after pipeline runs)
  │
  ▼
ETL (etl_local.py)
  Extract text from PDF
  Clean and normalise
  Save as JSON
  │
  ▼
Indexing (indexing.py)
  Embed with all-MiniLM-L6-v2
  Build FAISS flat-L2 index
  Write insurance_faiss.index   →   data/insurance_faiss.index
  Write insurance_metadata.json →   data/insurance_metadata.json
  Write kb_version.txt          →   data/kb_version.txt
  │
  (NER training is separate —
   not consumed by this workflow)
                                      │
                                      ▼
                                 src/ingestion/document_loader.py
                                   Loads FAISS index + metadata at startup
                                   Exposes retrieve(query, top_k) → [RetrievedDoc]
                                   Reports kb_version in every API response
```

**insurance-nlp-aws** owns: document intake, PDF extraction, text cleaning,
NER training, FAISS index construction, S3/Lambda/SageMaker deployment.

**This workflow** owns: consuming the index, ranking retrieved documents
against an assessment query, passing evidence to the LLM, and storing
`kb_version` in the audit record.

Neither repo contains the other's logic. The interface between them is two
files: `insurance_faiss.index` and `insurance_metadata.json`.

---

## How to rebuild the index

When new policy documents are available, rebuild the index in
`insurance-nlp-aws` and copy the output artifacts here:

```bash
# In insurance-nlp-aws/
python run_pipeline.py --local          # ETL + index, no AWS required
# or full AWS pipeline:
python run_pipeline.py

# Then copy to this repo:
cp insurance_faiss.index  ../insurance-ai-decision-workflow/data/
cp insurance_metadata.json ../insurance-ai-decision-workflow/data/
echo "faiss-$(date +%Y-%m-%d)" > ../insurance-ai-decision-workflow/data/kb_version.txt
```

After copying, restart the workflow API so `DocumentLoader` picks up the
new index. The `kb_version` on all subsequent cases will reflect the new
index build date.

## What the index currently contains

The active index was built from the synthetic insurance document set
generated by `insurance-nlp-aws/scripts/generate_data.py`. It covers:

- Auto insurance claims
- Property damage claims
- Medical / bodily injury claims
- Policy terms and coverage guidelines

Vector count: reported by `GET /health` → `subsystems.retrieval.doc_count`.

## Why Option A (companion repo) was chosen

`insurance-nlp-aws` contains AWS-specific deployment infrastructure
(Lambda packaging, SageMaker processing, S3 event triggers, CloudWatch
alarms, IAM role definitions). That infrastructure concern belongs in a
dedicated repo — it is not part of the underwriting decision workflow.

Pulling the AWS tooling into this repo would conflate two separate concerns:
document pipeline operations and underwriting decision logic.

The current arrangement keeps both repos independently deployable and
maintainable while making the data dependency explicit and documented here.
